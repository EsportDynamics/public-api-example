Webhook Monitor

Rankacy emits webhook events when demo and highlight processing reaches a terminal state. This page documents the receiver endpoint, delivery semantics, and displays the local event log.

Receiver Configuration

Endpoint POST /webhooks/rankacy
Deduplication X-Event-Id first, JSON id fallback
Typical Tunnel ngrok http 9000
Event types
EVT demo.processed.success
EVT demo.processed.failed
EVT highlight.processed.success
EVT highlight.processed.failed

Delivery Behavior

Semantics At-least-once delivery
Success Signal Any 2xx response
Expected Headers X-Event-Id, X-Event-Type, X-Request-Id
Local delivery note

Rankacy cannot deliver webhook requests to localhost directly. If you want to receive real events while running this app locally, expose it through a public HTTPS tunnel such as ngrok http 9000 and register that tunnel URL instead.
